## Directory Purposes

**`src/`:**
- Purpose: All source code for the application
- Contains: React components, utilities, styles, assets
- Key files: `main.jsx` (entry point), `App.jsx` (routes), `index.css` (global styles)

**`src/components/`:**
- Purpose: All React components organized by function
- Contains: Page section components (Hero, Pricing, FAQ, etc.) and UI primitives
- Key files:
  - Page components: `Hero.jsx`, `Pricing.jsx`, `FAQ.jsx`, `WaitlistPage.jsx`, `Navbar.jsx`, etc.
  - UI primitives: `ui/button.jsx`, `ui/input.jsx`, `ui/card.jsx`, `ui/accordion.jsx`

**`src/components/ui/`:**
- Purpose: Reusable, unstyled UI primitives following shadcn/base-ui pattern
- Contains: Base component wrappers with CVA variant system
- Key files: `button.jsx` (styled Button), `input.jsx` (styled Input), `card.jsx` (Card layout), `accordion.jsx` (Accordion component)
- Pattern: Each component wraps a @base-ui/react primitive and applies Tailwind classes

**`src/lib/`:**
- Purpose: Shared utility functions
- Contains: Helper functions and shared logic
- Key files: `utils.js` (cn() function for class merging)

**`src/styles/`:**
- Purpose: Global CSS variables and design tokens
- Contains: CSS custom properties, theme configuration
- Key files: `variables.css` (if used; most styling in index.css)

**`src/assets/`:**
- Purpose: Images and static media
- Contains: Hero images, SVG icons, etc.

**`public/`:**
- Purpose: Static files served at root (favicon, robots.txt, etc.)
- Committed: Yes
- Generated: No

**`dist/`:**
- Purpose: Production build output
- Generated: Yes (by `npm run build`)
- Committed: No (in .gitignore)

## Naming Conventions

**Files:**
- PascalCase for components (e.g., `Hero.jsx`, `WaitlistPage.jsx`, `Navbar.jsx`)
- camelCase for utilities (e.g., `utils.js`)
- kebab-case for CSS files (e.g., `variables.css`, `index.css`)
- Lowercase for directories (e.g., `components/`, `lib/`, `styles/`)

**Directories:**
- Lowercase, singular or plural based on content type
- `components/` (group), `ui/` (subgroup), `lib/` (utilities), `styles/` (CSS)

**Functions & Variables:**
- camelCase for all JavaScript identifiers (e.g., `handleEmailSubmit`, `setSubmitted`, `fadeInUp`)
- UPPERCASE for constants (e.g., `FOUNDING_PRICE = 79`, `SPOTS_PER_MARKET = 5`)
- Underscore prefix for internal/private functions (rarely used; not a strict convention observed)

**Types/Components:**
- PascalCase for component names (e.g., `function Hero()`, `function WaitlistPage()`)
- PascalCase for exported UI primitives (e.g., `function Button()`, `function Input()`)

**CSS Classes:**
- Tailwind utility classes (e.g., `px-4`, `text-orange`, `md:px-6`)
- Custom Tailwind theme colors (e.g., `text-charcoal`, `bg-orange`, `text-light-bg`)
- Standard shadcn color tokens (e.g., `bg-primary`, `text-foreground`)

## Where to Add New Code

**New Feature (Page Section):**
- Primary code: `src/components/NewFeature.jsx` (or `src/components/NewSectionName.jsx`)
- Add import to `src/App.jsx` in appropriate route or to `LandingPage` for new sections
- Tests: None (not part of current project structure)
- Example: To add a new "Testimonials" section, create `src/components/Testimonials.jsx` and add it to LandingPage between `LeadTypes` and `Pricing`

**New Route/Page:**
- Implementation: `src/components/NewPage.jsx` (or `src/pages/NewPage.jsx` if creating pages directory)
- Register in `src/App.jsx`: Add `<Route path="/new-page" element={<NewPage />} />`
- Any shared page layout goes in App.jsx or LandingPage wrapper

**New UI Component (Primitive):**
- Implementation: `src/components/ui/newcomponent.jsx`
- Follow pattern in `button.jsx` or `input.jsx`:
  1. Import base component from `@base-ui/react/{component}`
  2. Define CVA variant schema (if applicable)
  3. Export wrapper component that accepts className and merges with variants using `cn()`
- Import from `src/components/ui/` in feature components

**New Utility or Helper:**
- Shared helpers: `src/lib/newhelper.js`
- Import in components: `import { helperFunction } from '@/lib/newhelper'`
- Example: If adding form validation helpers, create `src/lib/validation.js`

**New Styles/Tokens:**
- Design tokens: `src/index.css` (in `@theme inline` block)
- Component-specific styles: Use Tailwind utilities in JSX `className` (not separate CSS files)
- No separate CSS files for components; prefer utility-first approach

## Import Path Aliases

**`@/`:** Maps to `src/` directory
- Usage: `import { cn } from '@/lib/utils'`
- Definition: `vite.config.js` line 10: `alias: { '@': path.resolve(__dirname, './src') }`
- Why: Cleaner imports; no relative path hell (`../../..`)
